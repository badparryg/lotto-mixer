# Lotto Project Codex Rules

## 자동 배포 원칙

- 사용자가 `중단`, `보류`, `배포하지마`, `로컬만`이라고 말하지 않는 한 코드 수정 작업은 로컬 검증 통과 후 자동으로 배포까지 완료한다.
- 더 이상 정확한 `배포진행해줘` 문구를 기다리지 않는다.
- 배포 후에는 실제 배포 페이지를 열어 변경사항이 반영됐는지 확인한 뒤 결과를 보고한다.
- destructive 명령, 데이터 파일 직접 변경, 추천/당첨 검증 알고리즘의 고위험 변경, 원격 롤백이 필요한 실패 상태에서는 자동 배포를 멈추고 상태를 보고한다.

## 기본 완료 기준

- `npm run check`
- `npm test`
- 배포 페이지 smoke check

## 금지 영역

- `data/**`
- `build/**`
- `node_modules/**`
- 사용자 요청 없는 추천 알고리즘 로직 변경
